Lightgrid

Version 2.0.0
Latest version published on 10/28/14

A simple, fluidic and responsive 12 column grid system for css.

What's new

  • Almost a whole rewrite of the library
  • Added less css support
  • Updated to reflect the latest normalize.css changes
  • Improved typography
  • Removed the need to add .last class for the lsat column

Lightgrid css will divide your page into 12 columns. You can arrange the divs to take up column space by adding col-* classes.
Previously you needed to add the .last class in the last column. Now you don't need to do that.


Try it now

First, add lightgrid.css in your html file.
Add <div class="row"></div> in your file.
Example:
<div class="row">
<div class="col-5">
col-5
</div>
<div class="col-5">
col-5
<div class="col-2">
col-2
</div>
</div>
This will add up 5 + 5 + 2 to make a 12 column grid.

Extras

Offsetting

You can also offset a div by any number of columns.
You can do this by adding offset-* in your div class.

Example:
<div class="row">
<div class="col-4 offset-4">
col-4 offset-4
</div>
<div class="col-2 offset-2">
col-2 offset-2
</div>
</div>

col-4 offset-4
col-2 offset-2 last

Images

Images are responsive and resize according to the parent element's size.

col-6 last

Responsive

At 768px (ipad width), lightgrid will automatically make all divs to 100% width.
Go ahead, resize this window to see it in action.

Less css support

The latest version of lightgrid is made in less css. In case you want to build from source, drop the less folder from the lightgrid repository into prepos or any other less compiler. Start modifying your build by editing the config.less file and remove anything you don't need.
Also, take a look under the hood and remove the imports you don't need in your site. For example, if you are already using a reset or normalize css then you can remove the import statement for normalize.less from the lightgrid.less file.

Browser compatibility

Lightgrid is fully supported in Ie7 and above.
